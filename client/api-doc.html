Location Station Map

Welcome to Location Station! Here locations are stored only using their associated latitude and longitude.


On the map, you can click any place to create a waypoint. Click this waypoint once more to see it's latitude and longitude. If you so desire, you may add this location to the API, by giving it a name (required) and a description (optional) after clicking the "Add Location" button.


You can search for locations by name in the sidebar in the homepage. You can also add a location at the bottom of the sidebar, but you will have to manually input the latitude and longitude of your desired location.


You can see a list of all locations in the "Locations" page. Here you can also edit locations to change their name, latitude, longitude and/or description, by clicking on the location's "Edit Location" button.

Final Submission Questions


What dataset are you using?

I decided to use my own dataset. Though very limited in number and a bit barebones in data for each object, I had to use a custom one since my project was very unique. I made sure that the data that was stored has a nested data, specifically the date and time that the locations were created. This project is meant to be more like a sandbox, where users can explore the map and add their locations, as opposed to something that would be used as a reference.



What went right in the development of this project?

Specifically the modals that I created were smooth and are easy for users to interact with. Since I was using mapbox, their documentation (https://docs.mapbox.com) was really easy to follow and understand. Custom styling for the map as well as 3D visualization of buildings did not take as long as I originally anticipated to implement.



What went wrong in the development of this project?

Many things went wrong in this project. Apart from the extremely late submission due to sickness and other assignments in different classes, I've come to realize an important problem that I have. Though this is more of a personal issue, I tend to have a problem where I overscope a project too far. I think of too many features and at some point it becomes a heavy burden. But I couldn't back out anymore, not after all the effort I already put in. This has been a recurring issue for some time. Regardless, I am both proud and dissapointed in myself, proud that I was able to pull through and actually complete this assignement, since it look a lot of time and a lot of struggle. On the other hand, I am dissapointed that I wasn't able to complete it on time.



What did you learn while developing this project?

I've learned many things, code wise, personal wise and design wise. When it comes to code, I would say I've more familiarized myself with the code. Practice is king and after looking over many areas of code and finding bugs, understanding what data goes where and how it's handled, I would say I've become much more proficient. Personal wise, I've learned that it's best to start small and build from there. This makes my webpage feel less like an API and more like a dynamic, interactable piece of software. Though the level of responsiveness is limited and I was never able to implement the delete buttons for each location in the location.html page, the functionality that is currently present, at least in my opinion, is still impressive.



If you used any borrowed code or code fragments, where did you get them from? What do the code fragments do? Where are they in your code?

Mapbox is the main star of my project. I used it not only to visualize the locations, but also created a way to add locations using it. I relied on the documentation as mentioned earlier, using many code fragments, which can be found in the index.html <"script"> tag, near the beginning. This includes setting specific colors for different roads, parks, landmarks, etc., but also things like setting zoom, flyto, layers and clicking mechanics all come from the mapbox documentation. All other code was written by me.
